📖 Важное пояснение: <label> vs <p>

<label> - это специальный элемент, который связывается с полем ввода через атрибуты for и id. Когда пользователь кликает на текст label, фокус автоматически переходит на связанное поле. Это улучшает удобство использования форм.

<p> - это обычный абзац текста. Он не имеет связи с полями ввода и не реагирует на клики. Используйте label для подписей к полям форм!

Пример связки: <label for="username">Имя:</label> <input id="username">

🎯 Пример формы с method="POST"

🔍 Разница между GET и POST:
• GET - данные добавляются в адресную строку (видно все параметры). Подходит для поиска, фильтров.
• POST - данные отправляются скрыто в теле запроса. Используйте для паролей, файлов, личной информации.

📐 ЧАСТЬ 2: ВЫРАВНИВАНИЕ ЭЛЕМЕНТОВ В CSS

Ниже показаны 5 основных способов центрирования блоков. Каждый способ имеет свои особенности и применяется в разных ситуациях.

1️⃣ Способ: text-align: center

📌 Я выровнен с помощью text-align: center
(display: inline-block)
⚙️ Как это работает:
• Родителю задаётся text-align: center
• Дочерний элемент должен иметь display: inline-block или display: inline
• Почему так: text-align управляет выравниванием только строчных и строчно-блочных элементов. Обычные блочные элементы (div с display: block) занимают всю ширину, поэтому центрирование не сработает.
• Когда использовать: для центрирования текста, иконок, небольших блоков внутри текста.

2️⃣ Способ: margin: auto

📌 Я выровнен с помощью margin: 0 auto
(ширина задана: 300px)
⚙️ Как это работает:
• Блоку задаётся фиксированная width (ширина)
• Применяется margin-left: auto и margin-right: auto (или margin: 0 auto)
• Браузер автоматически вычисляет равные отступы слева и справа, центрируя блок
• Важное условие: ширина блока должна быть меньше ширины родителя!
• Уточнение: По спецификации CSS, margin: auto по вертикали всегда вычисляется как 0, потому что браузер не может предсказать высоту динамического контента.
• Как исправить? Используйте Flexbox
• Когда использовать: для центрирования блочных элементов (картинок, карточек, меню) с известной шириной.

3️⃣ Способ: absolute + relative + transform

📌 Я выровнен с помощью position: absolute
left: 50%; top: 50% + transform: translate(-50%, -50%)
⚙️ Как это работает:
• Родителю задаётся position: relative (создаётся точка отсчёта)
• Дочернему блоку задаётся position: absolute (элемент выпадает из обычного потока)
• left: 50% - смещает левый край дочернего элемента на 50% от ширины родителя
• transform: translate(-50%, -50%) - сдвигает элемент на половину его собственной ширины и высоты
• Результат: элемент оказывается точно по центру родителя и по горизонтали, и по вертикали
• Когда использовать: для модальных окон, всплывающих подсказок, элементов поверх другого контента.

4️⃣ Способ: Flexbox (рекомендуемый для начинающих)

📌 Я выровнен с помощью Flexbox!
display: flex; justify-content: center; align-items: center;
⚙️ Как это работает:
• Родителю задаётся display: flex (включается flex-режим)
• justify-content: center - центрирует по главной оси (обычно по горизонтали)
• align-items: center - центрирует по поперечной оси (обычно по вертикали)
• важное "НО"! : когда мы меняем направление flex контейнера через свойство
flex-direction: (column/row/row-reverce/column-reverce)направление этих осей так же меняется
попробуйте поэкспериментировать с этим, на этом сайте есть интерактивные примеры
• Преимущества: не нужно знать размеры элементов, работает идеально во всех браузерах, просто в понимании
• Когда использовать: это самый универсальный и современный способ! Используйте его для большинства задач центрирования.

5️⃣ Способ: CSS Grid Layout

📌 Я выровнен с помощью Grid!
justify-self: center; align-self: center;
⚙️ Как это работает:
• Родителю задаётся display: grid (включается сетка)
• grid-template-columns: 1fr - создаётся одна колонка
• Дочернему элементу задаётся justify-self: center - центрирование по горизонтали внутри ячейки
• align-self: center - центрирование по вертикали внутри ячейки
• Можно использовать сокращение: place-self: center
• Когда использовать: для сложных сеток и когда нужно разное выравнивание для разных элементов внутри одной сетки.

🎨 Дополнительный пример: разные варианты выравнивания в Grid

Элемент A
justify-self: center
align-self: center
Элемент B
justify-self: start
align-self: start
Элемент C
justify-self: end
align-self: end
📌 Пояснение: В этом примере показано, как каждый элемент может иметь своё собственное выравнивание внутри сетки. justify-self управляет горизонтальным положением, align-self - вертикальным. Это очень удобно, когда элементы в одной сетке должны быть расположены по-разному.
